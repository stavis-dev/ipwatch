
# ipwatch

Минималистичная кроссплатформенная консольная утилита на Go для отслеживания вашего внешнего IP-адреса, определения провайдера (ISP) и ведения локальной истории адресов с комментариями.

Программа поставляется в виде одного бинарного файла, не имеет внешних зависимостей и работает одинаково хорошо на Linux, Windows и macOS.

## Возможности

* **Быстрый режим (по умолчанию):** Моментальное получение текущего IP без задержек.
* **Подробный режим (`-t`):** Вывод красивой таблицы с текущим IP, названием провайдера (ISP) и списком прошлых совпадений, если этот IP уже использовался.
* **Логирование:** Сохранение IP в локальный CSV-файл (разделитель — табуляция) без комментариев или с кастомной заметкой.
* **Нормализация ISP:** Автоматическая замена длинных технических названий провайдеров на короткие и понятные (настраивается через конфиг).
* **Кастомный IP (`-i`):** Возможность проверить и сохранить конкретный IP-адрес вручную.

## Использование

### 1. Быстрая проверка текущего IP
Выводит только ваш текущий IP-адрес и статус (новый или уже встречался ранее).
```bash
./ipwatch

```

### 2. Подробный вывод в виде таблицы

Запрашивает данные у расширенных провайдеров и выводит полную информацию, включая ISP. Если этот IP уже есть в логах, утилита покажет таблицу с историей его прошлых использований.

```bash
./ipwatch -t

```

### 3. Сохранение IP в лог

Сохранить текущий IP (использует быстрый чек):

```bash
./ipwatch -s

```

Сохранить текущий IP с текстовым комментарием (автоматически включает подробный режим для определения провайдера):

```bash
./ipwatch -c "Работаю из коворкинга"

```

### 4. Просмотр всей истории

Вывести в терминал красивую таблицу со всеми сохраненными записями:

```bash
./ipwatch -l

```

### 5. Проверка конкретного IP вручную

```bash
./ipwatch -i 8.8.8.8

```

## Конфигурация

Утилита работает "из коробки" со стандартными настройками, но вы можете кастомизировать пути и правила нормализации провайдеров через файл `ipwatch.json`.

Узнать, где утилита ожидает увидеть файл конфигурации на вашей системе:

```bash
./ipwatch --cfg-path

```

### Пример файла `ipwatch.json`:

```json
{
  "storage": {
    "log_path": "~/Documents/my_ip_history.csv"
  },
  "isp_normalization": {
    "PJSC MegaFon": "MegaFon",
    "International Hosting Company Limited": "NUXT",
    "Mobile TeleSystems": "MTS",
    "16345 Mobile Region": "Beeline"
  }
}

```

## Пути по умолчанию (без конфига)

* **Файл логов (`ipwatch_log.csv`):** * Windows: Папка `Документы` (`%USERPROFILE%\Documents`)
* Linux/macOS: Папка `~/Documents` или скрытая директория `~/.ipwatch/` (в случае отсутствия папки Документов).
Узнать точный путь к текущему логу: `./ipwatch --log-path`


* **Файл конфигурации (`ipwatch.json`):**
* Windows: `%APPDATA%\ipwatch\ipwatch.json`
* macOS: `~/Library/Application Support/ipwatch/ipwatch.json`
* Linux: `~/.config/ipwatch/ipwatch.json` (согласно XDG)

## Коды возврата (Exit Codes)

Утилиту удобно использовать в автоматических сценариях (например, в bash-скриптах или cron):

* `0` — Скрипт отработал успешно, текущий IP **новый** (ранее не встречался в логах).
* `1` — Произошла ошибка ИЛИ текущий IP **уже использовался ранее**.
